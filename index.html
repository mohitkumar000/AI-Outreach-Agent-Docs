AI Outreach Agent: Technical Specification & Implementation Plan

Version 1.1 | Date: September 6, 2025

Primary Stakeholders: Rishi/Team (Operator), Development Team

Stack: React.js (Frontend) | Spring Boot (Backend) | MySQL (Database) | Redis (Queue/Cache)

Contents

Goal & KPIs Scope Architecture Database Workflow AI & Emails Deliverability QA Checklist Team Work

1. Goal & Core Metrics

Project Goal: Automate sending personalized cold emails to AI companies/startups, covering the full outreach cycle (lead intake → emails → replies → reporting).

  • Daily Send Volume: ~2500 emails
  • Hard Bounce Rate: < 3%
  • Positive Reply Rate: > 10%
  • Time-to-first-send: < 60 mins after CSV upload
  • Personalization latency: < 7s per lead (p95)

2. Scope of Work

MVP (2–3 weeks)

  • Lead sourcing & enrichment
  • Validation (MX check)
  • Email sequences with warmup
  • Reply/link tracking
  • Simple dashboard + CRM sync

Phase 2

  • LinkedIn DM drafts
  • AI lead scoring & smart replies
  • Calendly booking automation
  • A/B testing + inbox rotation

3. System Architecture

Built with React (frontend), Spring Boot microservices (backend), MySQL (DB), Redis (queue/cache), S3 (storage). Observability with OpenTelemetry + Grafana.

Architecture Diagram

4. Database Design

Core tables: Leads, Campaigns, Messages, Events, Suppression. These hold info about contacts, campaigns, sends, and interactions.

5. End-to-End Workflow

  1. Lead Intake: React upload → Spring Boot API → Redis job → Enrichment.
  2. Enrichment: MX check, fetch website, AI personalization.
  3. Sequencing: Quartz scheduler → Select leads → Push Email jobs to Redis.
  4. Sending: Spring Mail → SMTP inboxes with tracking links/pixels.
  5. Tracking: Opens/clicks/replies logged in events table.
  6. Triage: AI classifies replies → Pause/reschedule as needed.
  7. Handover: Push interested leads to HubSpot/Sheets.
  8. Reporting: Analytics service → Dashboard charts (Recharts).

6. AI Personalization & Email Sequence

AI helps with writing personalized intros, pain points, and CTAs. Prompts instruct the LLM to output JSON with icebreaker, pain_hypothesis, and CTA.

{
  "icebreaker": "string (<= 35 words)",
  "pain_hypothesis": "string (<= 25 words)",
  "cta_variant": "string (one line, specific next step)"
}

Email Templates:

  • Day 0: Main pitch + pilot idea.
  • Day 3: Short nudge + proof point.
  • Day 9: Breakup email with resource link.

7. Deliverability & Compliance

  • SPF, DKIM, DMARC setup checks
  • Per-inbox limits + warmup mode
  • One-click unsubscribe → suppression table
  • Admin: suppression import, blocklists, human-approve toggle

8. QA Checklist

  • MX + SMTP validation works
  • DKIM/SPF/DMARC verifier passes
  • Events (open/click/reply) visible in dashboard
  • Unsubscribe link functional
  • AI personalization fields filled correctly
  • Timezone-aware sending + holiday skip works
  • Deduplication prevents duplicates

9. Team Work Split

Mohit

Frontend – React UI (forms, dashboard, charts).

Neerav

Backend – Spring Boot APIs, email logic, security.

Rudransh

AI – LLM prompts, lead scoring, smart reply drafts.

Rishi

Documentation & Marketing – Guides, docs, outreach plans.

Shared

Testing, deployment, DB schema design, architecture decisions (done together).
