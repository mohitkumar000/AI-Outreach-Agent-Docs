1. Goal & Core Metrics

Project Goal: Automate sending personalized cold emails to AI companies/startups.

  • Emails sent daily: ~1,000
  • Response rate target: 10â€“15%
  • Spam complaint rate: < 0.1%
  • Delivery rate: > 98%

2. Scope

  • In Scope: Lead import, segmentation, campaign creation, email sending, reply detection, deliverability tracking.
  • Out of Scope: Real-time dashboards, multi-channel (LinkedIn/SMS).

3. High-Level Architecture........

System uses clear separation between frontend, backend, and database.

  • Frontend (React.js): Campaign setup, dashboards.
  • Backend (Spring Boot): Orchestration, campaign logic.
  • Database (MySQL): Leads, campaigns, messages, events.
  • Queue (Redis): Email scheduling.

4. Database Design

Tables needed:

  • Leads: Contact details, scoring, status.
  • Campaigns: Target audience & email sequence.
  • Messages: Outbound emails.
  • Events: Opens, clicks, replies.
  • Suppression: Blocked/unsubscribed emails.

5. Workflow

  1. Leads imported via CSV/API â†’ stored in MySQL.
  2. Campaign defines audience, timing, templates.
  3. Backend schedules emails via Redis.
  4. SMTP provider sends email.
  5. Webhooks log opens, clicks, replies.
  6. Suppression applied automatically.

6. AI & Email Generation

LLM (AI) used for personalization:

  • Input: lead data + company context.
  • Output: subject line + email body.
  • Stored in DB before sending.

7. Deliverability Practices

  • Multiple inboxes for load distribution.
  • Warmup new domains.
  • Rotate sending IPs.
  • Track spam complaints.

8. QA Checklist

  • Email uniqueness.
  • Opt-out link working.
  • Bounce rate < 5%.
  • Threaded replies.
